# 让 AI 知道现在几点、外面什么天、你手机用了多久

> 含一个很多人搜不到的方法：**iPhone 的屏幕使用情况怎么同步到 Mac、又怎么读出来。**

想让网页或后端里的 AI 不再活在真空里——知道**现在几点**、**你隔了多久才回它**、**窗外什么天**、甚至**你今天手机刷了多久**。

办法其实很朴素，一句话就能说清：

> **不是让 AI 自己去查，而是你提前把这些算好、拼成一小段文字，塞进同一次对话的提示词里。**

AI 并不会、也不需要自己去查——它只是在提示词里「读到」了「现在是晚上 11 点」「外面在下雨」这几行字而已。它们搭着用户的消息一起发出去，对模型来说就是一次普通对话，不会多产生任何查询请求。

---

## 拆成几块，需要哪块抄哪块

| 这块 | 它回答什么 | 数据从哪来 |
|---|---|---|
| 时间 | 现在几点、星期几、深夜还是清晨 | 浏览器本地 `new Date()` |
| 间隔 | 用户隔了多久才回 | 翻聊天记录里上一条的时间戳 |
| 天气 | 用户那边什么天 | open-meteo 免费接口，浏览器直接问 |
| 心情 | 用户此刻亲手记的状态 | 浏览器本地存的状态，只取最近 12 小时 |
| 手机 | 电量、正用着哪个 app、今天刷了多久 | iOS 快捷指令上报 + 读 Mac 上的 Biome 文件 |

---

## 最难的那块：iPhone 的屏幕使用情况怎么到 Mac、又怎么读

有两种办法，建议都上、互相补位。

**办法一（主动、实时、最准）**：iPhone 用「快捷指令 + 个人自动化」，把电量、当前 app 发到你后端的一个简单接口。这是用户真的按下去的那一下，准。

**办法二（自动、被动、有延迟）**：macOS 其实已经悄悄把 iPhone 的「当前前台 app」同步到 Mac 上了，藏在这个路径——就是大家常常找不到的那个文件：

```
~/Library/Biome/streams/restricted/App.InFocus/remote/<每台设备一个编号>/<一堆小文件>
```

为什么大家找不到：

- `Biome` 是 macOS 记录设备活动的机制（屏幕使用时间、智能建议背后用的就是它）；
- `App.InFocus` 记的正是「当前在前台的是哪个 app」；
- 关键在最后那个 **`remote`**——`local` 是本机的，**`remote` 装的才是从你别的设备（iPhone）同步过来的**；
- 网上老教程只盯着旧的 `knowledgeC.db`，而新系统这条实时记录走的是 Biome，按老路自然找不到。

**让这个文件有内容的三个前提（缺一不可）：**

1. iPhone 和 Mac 登录同一个 Apple ID、都开着 iCloud；
2. iPhone：设置 → 屏幕使用时间 →「跨设备共享」打开；
3. 读它的程序要有「完全磁盘访问」权限（这个目录是受保护的）。

**注意它天生有延迟**：靠 iCloud 同步、慢一拍，还会整点把睡前旧记录一批批补传上来。所以要配三步过滤——先确认用户真醒着、用真实操作当锚点筛掉旧记录、被动读到的只悄悄存底不拿去主动发通知。完整代码和过滤逻辑都在 [`index.html`](./index.html) 里。

---

## 最后：拼进提示词时顺手省钱

把「整段对话都不变的」（人设、系统提示）放进**会被缓存的部分**；把「每句都在变的」（时间、天气、心情、手机状态）全挪到**缓存外面**单独一段。否则时间每分钟都在变，会把整块缓存顶作废，钱就白花了。

每一段都带一句「该怎么用」的提示——「合适时自然带一句、别每次都念」。**你给的是背景信息，不是命令**，把判断权交还给模型，它才会像个活人。
